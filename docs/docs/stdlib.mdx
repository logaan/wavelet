# Standard library

`wavelet:std/core` is implicitly imported and open in every file. Because identifiers are WIT labels, there is no `+` or `<` Рђћ the names are short and boring, a deliberate cost of "the syntax is WAVE, no exceptions".

Every function below is **implemented and runnable today**. Most take their arguments as a list or tuple payload, so `add[2 3]` and `add(2, 3)` are equivalent; a few take a single value. Edit any example and press Run.

:::note[What "function" means here]
These are the core *builtins*. Convenience **macros** (`And`, `Or`, `TryLet`) are **not** shipped Рђћ define them yourself with [`DefMacro`](./language/macros). Two pre-bound *values* also live in core: `none` (the empty option) and `pi`.
:::

## Cells

A `cell` is the standard library's mutable reference Рђћ the one place mutation lives, since [the core has none](./language/evaluation#no-loops-no-mutation-no-early-return).

| Function | Shape | Effect |
|---|---|---|
| `cell-new` | `cell-new(v)` | a new cell holding `v` |
| `cell-get` | `cell-get(c)` | read the current value |
| `cell-set` | `cell-set[c v]` | overwrite the value (returns unit) |
| `drop` | `drop(h)` | release a resource handle |

<Playground id="std-cells" />

:::info[Resources beyond cells]
`cell` is the only resource type that ships today. General resource handles Рђћ opening, methods, owned-handle drop semantics Рђћ are part of the component-interop story and are **not yet implemented**; `drop` currently accepts any value and returns unit. See [Components & composition](./components#resources) and the [roadmap](./roadmap).
:::
