# Reader sugar

Wavelet's tokens *are* [WAVE](https://github.com/bytecodealliance/wasm-tools/tree/main/crates/wasm-wave)'s tokens, and the reader is where all sugar dies: every surface form desugars to a canonical WAVE value before anything is evaluated. This page enumerates each reader-sugar form, one per section, with the canonical value it produces.

A form is **reader sugar** when it is *not already* a valid WAVE value, or when the reader **transforms** it into a different shape as it reads. That is the whole list below.

The plain WAVE structures are **not** sugar — they read straight through, unchanged:

| You write | Reads as | |
|---|---|---|
| `(a, b)` | `(a, b)` | tuple |
| `[a, b]` | `[a, b]` | list |
| `{k: v}` | `{k: v}` | record |
| `{read, write}` | `{read, write}` | flags |
| `foo`, `42`, `"hi"` | `foo`, `42`, `"hi"` | a bare case, an atom |

These are WAVE values to begin with; the reader has nothing to do to them. (A bare identifier *evaluating* to a variable reference is a rule of [evaluation](./evaluation), not of reading — as data it is just a payload-less case.) The single liberty Wavelet takes with all of them is making commas optional, which is the first piece of sugar below.

You can confirm any desugaring yourself with `wavelet read`, which prints the canonical WAVE tree for a piece of source:

```console
$ echo 'f(x y)' | wavelet read /dev/stdin
f((x, y))
```

## 1. Commas and newlines are whitespace

As in Clojure, commas are whitespace, and so are newlines. These all read identically:

```wavelet
[1 2 3]
[1, 2, 3]
[1,
 2,
 3]
```

```console
$ echo '[1 2 3]' | wavelet read /dev/stdin
[1, 2, 3]
```

This is the sugar that makes Wavelet a *superset* of WAVE rather than a different language: drop the commas and a WAVE value is already Wavelet. The canonical printer always puts the commas back, so anything Wavelet reads prints as strict WAVE and can be handed to any WAVE tooling. Because every comma-free tuple, list, record, and flag set is just this rule applied, none of them needs its own entry here.

## 2. Grouping parentheses

Parentheses around a *single* form are transparent: the reader strips them and keeps the form inside.

| You write | Canonical WAVE form |
|---|---|
| `(a)` | `a` |

```console
$ echo '(a)' | wavelet read /dev/stdin
a
```

This is genuine sugar — `(a)` is rewritten to `a` on read. A single-element parenthesised form is **not** a one-tuple; tuples need two or more elements, which read through unchanged.

## 3. The attachment rule (call forms)

Wavelet has exactly **one** whitespace-sensitive rule. A `(`, `[`, or `{` that *immediately* follows an identifier — with no intervening whitespace — supplies that identifier's payload. This is a call form. With whitespace in between, the bracketed expression is a separate, free-standing value:

```wavelet
delete-file{path: "foo.md" force: true}   // a call: payload is a record
delete-file {path: "foo.md" force: true}  // two forms: a variable, then a record
```

The bracket you attach chooses the payload shape, and most shapes are rewritten as they read:

| You write | Canonical WAVE form | |
|---|---|---|
| `f(x)` | `f(x)` | single value — already valid WAVE, unchanged |
| `f(x y)` | `f((x, y))` | two-plus values become a **tuple** payload |
| `f[x y]` | `f([x, y])` | a bracketed **list** payload |
| `f{a: 1 b: 2}` | `f({a: 1, b: 2})` | a braced **record** payload |
| `f()` or `f[]` | `f([])` | zero arguments become an empty list |

```console
$ echo 'f(x y)' | wavelet read /dev/stdin
f((x, y))
$ echo 'f[]' | wavelet read /dev/stdin
f([])
```

A bare `f(x)` is already a valid WAVE variant case, so the reader leaves it alone; every other shape is transformed — juxtaposed forms get wrapped into a tuple, a `[…]` or `{…}` becomes the case's single payload value, and an empty bracket becomes `[]`. As data, a call form is a variant case: `log("hi")` is the WAVE case `log` carrying a string. Evaluation gives these case-shaped values their meaning; under `Quote`, they are just data.

## 4. Qualified calls

A `/` between two names is a qualified reference: the name after the slash, taken from the import aliased by the name before it. WIT identifiers contain no slash, so this is a Wavelet-only token, and it attaches a payload exactly like an ordinary call:

| You write | Canonical WAVE form | |
|---|---|---|
| `kv/get{...}` | `kv/get({...})` | name `get` from the import aliased `kv` |

```console
$ echo 'kv/get{bucket: b}' | wavelet read /dev/stdin
kv/get({bucket: b})
```

## 5. Doc comments

`//` runs to the end of the line and is discarded — comments never reach the form tree. A *doc* comment — `///` — is the exception, and the sugar: instead of being thrown away, its text attaches to the following form as metadata, surfaced in `wavelet wit` output.

```wavelet
/// Delete a file from the working directory.
Def delete-file Fn {path: string} ...
```

Plain `//` comments leave no trace in the read tree; `///` doc comments ride along on the form they precede.

## 6. TitleCase macro sugar

A **TitleCase** identifier — mixed-case words, each starting with a capital (`If`, `DefMacro`, `TryLet`) — is reader sugar for a macro call. This can't collide with ordinary identifiers: WIT words are all-lower or all-UPPER, so TitleCase tokens are syntactically free. The token kebab-izes and gains a `-MACRO` suffix: `If` ↦ `if-MACRO`, `DefMacro` ↦ `def-macro-MACRO`.

A TitleCase head does **not** require parentheses. The reader looks up the macro's declared **arity** and consumes exactly that many following forms, collecting them into a tuple payload:

```wavelet
If eq[foo bar] log("match") log("nope")
```

```console
$ echo 'If eq[foo bar] log("match") log("nope")' | wavelet read /dev/stdin
if-MACRO((eq([foo, bar]), log("match"), log("nope")))
```

Three consequences:

1. **Define before use.** A macro must be in scope before it's read — a core form, defined earlier in the file, or imported. The reader processes a file top to bottom and always knows every visible macro's arity.
2. **Explicit payload overrides arity reading.** `Unquote(x)` and `If(c t e)` attach their payload directly. The fully explicit `if-MACRO((c, t, e))` is always available too, and is what macros emit.
3. **Nesting needs no delimiters** when the last argument is itself a macro: `Def run Fn {} If c a b` reads exactly as intended, since each TitleCase head consumes its own arity recursively.

Variadic macros take a single list argument — arity stays fixed, the list flexes (`Do [a b c]`).

## At a glance

Every reader-sugar form, in one table:

| # | You write | Canonical WAVE form | |
|---|---|---|---|
| 1 | `[1 2 3]` | `[1, 2, 3]` | commas/newlines are whitespace |
| 2 | `(a)` | `a` | grouping parens, stripped |
| 3 | `f(x y)` | `f((x, y))` | attachment rule, call payload shapes |
| 4 | `kv/get{...}` | `kv/get({...})` | qualified call |
| 5 | `/// text` | metadata on the following form | doc comment |
| 6 | `If c t e` | `if-MACRO((c, t, e))` | TitleCase macro |

The [Syntax & reading](./syntax) page covers the same ground as prose, and [`wavelet read`](../cli#read) prints the canonical tree for any source so you can check the desugaring for yourself.
