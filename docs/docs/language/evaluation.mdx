# Evaluation

There are four evaluation rules. That's the whole model.

1. **Atoms** — booleans, numbers, chars, strings, flags — evaluate to themselves.
2. **A bare name** evaluates to its binding in the current lexical environment. Wavelet is a **Lisp-1**: one namespace for everything. Unbound names are errors. Enum cases, imported functions, and constructors are all just bindings.
3. **A call form** — a tuple `(head arg…)` in evaluation position — evaluates its arguments, **bundles** them (0 ⇒ the empty tuple, 1 ⇒ that value, ≥2 ⇒ a tuple), and applies the value bound to `head`. Heads are names or qualified names; a literal tuple **value** comes from `Quote` or a builtin. To call a *computed* function value, use [`apply`](../stdlib#meta--reflection).
4. **Special forms and macros** are recognised before evaluation and follow their own rules — see [Special forms](./special-forms) and [Macros](./macros).

## Functions take exactly one value

This is the rule that makes everything else fit together. A call bundles its arguments into one value — a scalar (1 argument) or a tuple (≥2) — which may also be a record or list, and that's what gives calls their n-ary feel. The `Fn`'s parameter braces describe how to receive it:

- A **record** argument binds parameters by **name**.
- A **list** or **tuple** argument binds them by **order**.
- A **scalar** argument binds a sole parameter directly.

## No loops, no mutation, no early return

The core deliberately omits these. Iteration is [tail recursion](./tail-calls); mutable state lives in resources (the standard library's [cell](../stdlib#cells)); error propagation is a macro over [`Match`](./pattern-matching) (see the `TryLet` pattern under [Macros](./macros#error-propagation-without-early-return)).
