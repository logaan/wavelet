# F5 error mapping (trap codes and a form to source-span table)

Surface emit/wit compile errors verbatim (they are already actionable
Result<_, String>). For runtime traps, emit distinguishable trap codes per
failure class (div-by-zero, type mismatch, out-of-bounds) and resolve them
through a form->source-span table so the REPL prints source context
comparable to the interpreter's "eval error: ...". This is the whole of F5:
build the trap-code scheme and span table here.
