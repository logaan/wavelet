# Replace the interpreter with the compiler - plan index

Migrated from dev-notes/replace-interpreter/index.typ.

Objective: make the wasm compiler the execution engine for the
compile-time and command-line surfaces, so the tree-walking interpreter no
longer runs user programs there. The interpreter is not deleted - it is
retained as the differential-testing oracle (CLAUDE.md), and the
differential harness stays for the life of the project. "Replace the
interpreter" means replace it as an execution path, not remove it from the
tree.

Out of scope: the docs playground. It keeps running on the interpreter (via
`eval_snippet`, lib.rs:113). Compiling the compiler to wasm32 and running
snippets in-browser is no longer a goal.

Surfaces to move off the interpreter:
- Macro expansion (compile time), powered today by `interp.expand_once` for
  local DefMacro (expand.rs:136); the foreign macro guest `macrolib` is also
  the interpreter. -> Step 1.
- `wavelet repl`, powered today by `Interp` per line (repl.rs). -> Step 2.
- `wavelet run`, powered today by `Interp` over the build set (runner.rs).
  Outside the two steps but shares Step 2's machinery (synthesize an entry,
  compile, run via wasmtime); fold it into Step 2 or drop it for `build` +
  `wasmtime`. Tracked as a Step 2 rider.

Dependency order and recommended sequence: F4 (differential tests) first so
every later change is checked against the interpreter -> Step 1 (compiled
macros), since a REPL line may define and use a macro -> F1 (core+stdlib) ->
F2 (value reader) -> F5 (diagnostics) -> Step 2 (REPL).

Risk register (the load-bearing tradeoffs):
- Coverage is the whole game: the backend is a strict subset of the
  interpreter until F1 closes; budget effort into the standard library (in
  Wavelet) over a small core, not per-builtin special cases in emit.rs.
- Drawing the core/stdlib line: misclassify downward and emit.rs bloats
  with special cases; misclassify upward and the stdlib can't be expressed.
- Keep the correctness oracle wired up: the hazard is changing the backend
  before the comparison is automated - the emit_* tests only assert the
  module encodes, never that it runs or agrees. Land F4 first.
- Error fidelity is a planned workstream, not a hope (F5): the REPL UX
  regresses on errors unless failures are mapped back to source.
- Macro expand/quasi at compile time: Quote/Quasi and the expand builtin
  codegen are unimplemented (emit.rs:1392) and are the riskiest single
  piece (Step 1).
