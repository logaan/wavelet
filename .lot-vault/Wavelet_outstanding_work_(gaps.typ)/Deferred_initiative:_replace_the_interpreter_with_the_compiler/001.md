# Deferred initiative: replace the interpreter with the compiler

Goal: make the wasm compiler the execution engine for the compile-time and
CLI surfaces (macro expansion, `repl`, `run`) so the tree-walker no longer
runs user programs there. The interpreter is retained as the
differential-testing oracle. The browser playground stays on the
interpreter (out of scope).

Gating prerequisites, none yet done, from dev-notes/gaps.typ:

- GC first - without it a compile-and-run REPL leaks on every definition
  (see the top-level "Implement garbage collection" Thing, already migrated).
- Core / standard-library split (F1) - move map/filter/fold/range/zip/
  split/join/contains/apply/cell-*/to-* and float/char arithmetic out of
  per-builtin special cases and into a Wavelet standard library over a
  small irreducible core, imported by default with a per-file opt-out.
  (See the top-level "Split standard library from core language" Thing,
  already migrated - that is this same item.)
- Value marshalling + printing (F2), Differential harness (F4), and
  Diagnostic fidelity (F5) - see the children of the plan index below, where
  they're scoped in full.
- Compiled Quote/Quasi/Unquote/Splice + expand codegen - the riskiest
  single piece; see Step 1's first task below.

This initiative has its own detailed plan (formerly
dev-notes/replace-interpreter/) and the feasibility study behind it
(formerly dev-notes/drop-the-interpreter.md), both migrated as children of
this Thing.
