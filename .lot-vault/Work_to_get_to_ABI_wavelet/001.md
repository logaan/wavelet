# Work to get to ABI wavelet

Make the Canonical ABI representation wavelet's *internal* value representation,
so that lift/lower at component boundaries becomes a no-op (or a memcpy) instead
of a per-value conversion between tagged boxes and ABI layout.

Today every value in the wasm backend is an i32 pointer to a tagged box in
linear memory (`emit.rs`), and every boundary crossing converts: lift = ABI ->
boxes, lower = boxes -> ABI. That uniform representation exists because the
core is dynamically typed. The type system is further along than its comments
admit (Phase A checking + Phase C overload resolution in `check.rs`, Phase B
WIT-signature inference and functors in `wit.rs`), but it is still *gradual*:
anything it does not model is `Unknown` and needs the boxed fallback.

"ABI wavelet" = finish the walk from gradual to total. Every expression gets a
concrete WIT type at compile time; representation becomes type-directed; boxes
disappear except (possibly) for closures. The trade is the dynamic core: this
is a prototype-stage language, so this plan proposes dropping or radically
reshaping the dynamic features rather than carrying two representations
forever.

Children: one category per work area, each with concrete steps. The last
category lists the language features that have to be dropped, with the
reasoning spelled out.
