# Defunctionalize closures - or fence off a private representation

Decision step, radical option first: defunctionalization. Every Fn in a
component becomes a case in a generated variant type (discriminant = which
function, payload = its captures, all concretely typed); application becomes
one generated dispatch function per arrow type. Closures then ARE plain WIT
variants — first-class functions keep working, become ABI values, and can
even cross component boundaries, which TAG_FN never could. The costs: the
set of functions must be closed per component (it is — compilation is
whole-file), and callback-heavy code compiles to branchy dispatch.
Alternative if rejected: keep one private non-ABI representation for
closures only, confined inside the component — the goal is then "ABI except
functions" and the memory-management step must keep headers for this one
family.
