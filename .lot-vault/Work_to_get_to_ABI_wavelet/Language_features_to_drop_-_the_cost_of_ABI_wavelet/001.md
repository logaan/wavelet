# Language features to drop - the cost of ABI wavelet

The honest ledger. Each child is a feature of today's dynamic core that
cannot survive a total, monomorphic, ABI-native wavelet, with the reasoning.
The theme: every one of these exists because values carry tags at runtime;
remove the tags and the feature either moves to compile time, moves to the
tree type, or goes away. Prototype-stage language — nothing here is sacred,
and several of these are arguably improvements independent of the ABI goal.
