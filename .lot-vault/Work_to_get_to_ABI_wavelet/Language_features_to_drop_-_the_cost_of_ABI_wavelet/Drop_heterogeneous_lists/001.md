# Drop heterogeneous lists

Lists become genuinely list<t>. Canonical layout packs elements at one
static size, so a list mixing ints and strings has no representation. Mixed
"shapes" are what variants are for (list<my-variant> stays fully typed), and
mixed *code-like* data is what tree is for. The checker already hedges here
(heterogeneous elements degrade to Unknown "because lists of mixed shape
appear in quoted data") — quoting moving to tree removes that pressure.
