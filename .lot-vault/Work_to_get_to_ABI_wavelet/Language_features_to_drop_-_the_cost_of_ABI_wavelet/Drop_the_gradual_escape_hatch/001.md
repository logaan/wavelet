# Drop the gradual escape hatch

Today a program the checker cannot type still runs, and failures surface as
boundary traps. Under ABI wavelet, "cannot type" means "cannot represent",
so inference failure becomes rejection. This deletes design.md's stated
trade ("some errors surfacing at the edge rather than at the keystroke") in
the good direction, but it is still a drop: exploratory untyped scripts stop
being valid programs unless they type.
