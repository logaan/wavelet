# 6.1 Step 1: compiled macro components

Migrated from dev-notes/replace-interpreter/step-1-macro-components.typ.

Goal: every macro - local DefMacro and imported macro library - runs as
compiled wasm at expand time, so expand.rs needs no Interp. A macro is a
pure function from forms to a form; forms are exactly the values the
backend already boxes (TAG_TUP/TAG_VAR/TAG_REC/TAG_LIST/scalars,
emit.rs:44-54), so a macro body is "just a function" emit.rs can compile -
once a few compile-time forms and builtins are taught to the backend.

Current state: local macros interpret (expand_file evaluates DefMacro forms
and expands call sites via Interp::expand_once, expand.rs:136,
interp.rs:319; quasiquote is Interp::quasi, interp.rs:350). Foreign macros
are components, but the guest interprets - `macros: true` imports resolve
to a wavelet:meta/macros component run under wasmtime (macrodep.rs,
host.rs), but the guest macrolib.rs IS the interpreter compiled to wasm
("strategy A"). The boundary contract is fixed: `manifest() ->
list<(string, u32)>` and `expand(name, args: tree) -> result<tree,
string>`, with tree<->arena marshalling in meta.rs - keep this contract,
only change who implements `expand`.

Target: strategy B. A macro component is an ordinary compiled Wavelet
component that exports wavelet:meta/macros. No interpreter in the guest;
the only genuinely new codegen is the compile-time form machinery the
bodies use.

Risks: Quote/Quasi codegen is the crux and must exactly match
Interp::quasi depth semantics (mirror the interpreter test-for-test).
expand-inside-macro has no clean compiled story without a host-mediated
callback or a guest mini-expander - defer and document. Build-time cost of
compiling+instantiating a wasmtime component to expand local macros is far
heavier than in-process interpretation - cache per file/package, skip
macro-free files. gensym hygiene must stay deterministic and
collision-free across separately compiled macro invocations within one
build.

Exit criteria: expand.rs contains no Interp/expand_once call in production,
and the macro guest links no interpreter. interp.rs's expand_once/quasi
remain in the tree as the F4 oracle. Local and foreign macros expand
identically to the interpreter across the macro corpus (F4), including the
try-let and nested-quasi cases. cargo test green; macro-free builds incur
no macro-component cost.
