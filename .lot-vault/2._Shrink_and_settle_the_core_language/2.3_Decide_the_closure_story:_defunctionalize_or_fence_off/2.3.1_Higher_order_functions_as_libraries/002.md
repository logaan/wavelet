Answered and accepted (Logan, 2026-07-03). The requested write-up is updates
003/005/006 on the parent (lot:033iWUQfu2XoRGFBRkLq5R); the accepted answer to
this Thing's specific question — `map([a], a-b) -> [b]` defined in one file,
imported and called from another with a locally defined function — is:

- A *generic* combinator cannot be a component export at all (Wavelet is
  monomorphic; nothing generic survives into WIT), so cross-file combinators
  are **expand-time imports**: compile-time template expansions (5.7.1)
  delivered through the existing macro-component machinery (build.rs already
  compiles macro libraries into importable components). The combinator's body
  lands in the calling component, the local function never escapes, and
  per-component defunctionalization (5.8) covers it.
- A *monomorphic* function crossing a boundary for real (callbacks/handlers)
  is the resource-lifting extension, 5.8.1 — borrow-only single-`call`
  resources, riding on 4.5.
- Ergonomic dependency for combinator libraries: 7.4 (an import usable as
  both runtime dependency and macro library).
