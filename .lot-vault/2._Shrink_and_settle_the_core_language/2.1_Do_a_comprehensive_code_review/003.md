arrive with the review's findings. The question that IS decidable now is
sequencing and format:

**Question: does the review run now — before the 2.2/2.3/2.4 decisions are
finalized — and how do its language-change proposals reach you?**

Options:

1. **Run it now, before finalizing 2.2–2.5 (recommended).** The re-scope in
   update 002 exists precisely because the review's second half (language
   changes that simplify the implementation) is an *input* to those decisions —
   e.g. it can put numbers on what each 2.2 drop actually deletes from
   check.rs/interp.rs. Scope per 002: reader/lexer, expand, wit.rs, check.rs,

refactors and bug fixes just get done (with tests); each *language-change or
scope-cut proposal* is posted as its own update on this Thing in the same
question/options/recommendation shape as the 2.2 children, so you can yes/no
each one in place.
