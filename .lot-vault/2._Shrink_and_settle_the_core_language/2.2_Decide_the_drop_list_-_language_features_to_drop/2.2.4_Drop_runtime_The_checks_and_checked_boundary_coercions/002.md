**Question: does `The` become pure static ascription — deleting the
compiler-inserted runtime checks and checked boundary coercions — with `safely`
surviving only where genuinely dynamic data enters?**

Options:

1. **Drop (recommended).** `The` is static ascription, nothing more (it already
   is for literals). Checked coercions on wavelet-side values are deleted:
   under total types those values are correct by construction, so the checks
   can never fire. `safely` remains meaningful only at real dynamic entry
   points — data parsed with `read`, and values lifted from foreign components.
2. **Keep the checks behind an opt-in validation flag.** Same default semantics
   as option 1, with a debug mode that reinstates boundary checks for hunting
   miscompiles. Cost: a second codegen path to keep working forever, duplicating
   what goal 1's differential and conformance harnesses already do better.
3. **Status quo.** Runtime checks exist because values can be mistyped at
   runtime; once that's impossible they are pure dead weight on every boundary.

Recommendation: **option 1.** Note also that validating what a foreign
component hands us is the canonical ABI's job at lift time — wavelet doesn't
need its own second fence there.
