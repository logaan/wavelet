# 2.2.7 Drop first-class functions - only if defunctionalization is rejected

Contingent drop. If the defunctionalization decision goes the other way,
first-class functions shrink to references to top-level defs (no captured
environment), or disappear in favour of explicit variant-plus-dispatch
written by hand. Also permanently out (already deferred as Phase D): passing
an *overloaded* name as a value — an overload set has no single type, so no
single representation.
