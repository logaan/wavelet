# 2.2.5 Drop or confine runtime eval

read gives you a tree; today you can walk it into values and apply it. A
typed component cannot evaluate arbitrary runtime-constructed code, because
its type is data (tree), not a static signature. Options, in order of
radicalism: keep read purely as data (trees in, trees out — still enough for
data formats and macros); allow eval only in the dynamic-island component;
drop runtime eval entirely and lean on the compile-time expander. The REPL
stays fine either way — it rebuilds and recompiles a scratch component per
definition, which is compile-time evaluation.
