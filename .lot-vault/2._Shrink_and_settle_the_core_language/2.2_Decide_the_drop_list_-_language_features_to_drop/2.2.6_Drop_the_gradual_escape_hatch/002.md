**Question: does inference failure become outright rejection — i.e. no program
that fails to type ever runs?**

Options:

1. **Drop the hatch (recommended).** A program the checker cannot type does not
   compile. Errors move from boundary traps to the keystroke, which is the good
   direction; and under ABI wavelet "cannot type" already means "cannot
   represent", so the hatch isn't implementable without reintroducing boxes.
2. **Keep a scoped dynamic opt-in** (per file or per form). Requires a boxed
   representation for the dynamic regions plus coercions at their edges —
   maintaining the entire machinery being deleted, in service of the minority
   case.
3. **Interpreter-only escape**: untyped scripts run interpreted but can never
   compile. Forks the language into two dialects and conflicts with goal 6.

Recommendation: **option 1.** The honest cost is that exploratory untyped
scripts stop being valid programs. The mitigation is what goal 3 is building
anyway: inference good enough that annotations are rare, plus the REPL for
poking at things. If exploration still hurts after that, the remedy is the
2.2.5 dynamic-island conversation — not keeping untyped whole programs alive.
