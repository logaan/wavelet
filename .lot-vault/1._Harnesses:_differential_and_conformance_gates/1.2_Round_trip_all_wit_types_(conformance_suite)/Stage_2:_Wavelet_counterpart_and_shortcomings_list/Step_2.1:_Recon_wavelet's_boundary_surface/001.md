# Step 2.1: Recon wavelet's boundary surface

Establish exactly how a wavelet file targets the roundtrip:suite world.

Findings so far (from design.md §3/§6, src/witdep.rs, src/emit.rs, README):

- Imports resolve against WIT vendored under the project's `wit/deps/`
  (wkg convention), via src/witdep.rs -> emit::Dep. So the suite's WIT can be
  vendored at conformance/wavelet/wit/deps/roundtrip-suite/world.wit.
- Import form: `Import {pkg: "roundtrip:suite/values" as: v}`; calls are
  qualified `v/u8-rt(...)`. Resources arrive as constructors/methods
  (`kv/open(...)`, method call = first arg is handle).
- Exports: `Export {name: ... iface: ... params: ... result: ...}`;
  `is_external_iface` (iface containing ':') in emit.rs/wit.rs shows exporting
  into a foreign package's interface (e.g. `roundtrip:suite/values`) has a
  code path — needs a hands-on test.
- CLI: `wavelet build <file.wvl> [-o dir]` produces a component;
  `wavelet run` interprets. All WAVE literals for WIT values exist (variants
  `circle(1.0)`, enums as bound names, flags `{read write}`, records, options
  some/none + flat, results ok/err + flat, chars, tuples via Quote).

Still to verify hands-on: DefType for variant/enum/flags/record declarations,
whether wavelet can EXPORT a resource (counter) — gaps.typ says user-declared
resources are unimplemented — and what the synthesized world looks like
(wasm-tools component wit on the artifact).
