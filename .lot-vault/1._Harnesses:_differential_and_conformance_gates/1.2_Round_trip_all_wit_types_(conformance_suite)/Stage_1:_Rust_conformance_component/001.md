# Stage 1: Rust conformance component

Build the wavelet-agnostic WIT round-trip conformance component in a new
`conformance/` directory of the wavelet repo (self-contained crate, no wavelet
dependency).

Scope (all agreed in the parent Thing's interview):

- WIT package `roundtrip:suite` covering every feature on
  <https://component-model.bytecodealliance.org/design/wit.html>: all
  primitives, list, option, all four result forms, tuple, record, variant,
  enum, flags, resources (constructor, method, static, borrow — treated the
  same as everything else), type aliases, multi-param and zero-param/zero-result
  functions, and nested compounds (list<record>, option<variant>,
  result<tuple, enum>, list<list<u8>>, ...).
- The component both imports and exports the suite interfaces (symmetry), plus
  exports a `runner` interface `run: func() -> result<_, list<string>>`
  returning one message per failed check.
- Transform convention documented in WIT doc comments: ints +1 wrapping,
  floats +1.0, bool negated, char next scalar, string appends "!", compounds
  recurse elementwise, enums rotate, variants keep case and transform payload,
  flags toggle every member, bare result flips ok<->err.
- Two hard-coded seed tables selected at build time -> two wasm files
  (roundtrip-a.wasm, roundtrip-b.wasm) so callees can't hard-code responses.
- Self-test harness: compose caller+callee with `wac plug` in both seed
  pairings, execute with `wasmtime run --invoke`; validate the WIT with
  wasm-tools. A script drives it; a README documents the protocol for other
  implementations.

Deliverable: PR 1, with the self-test green (rust-a vs rust-b in both
directions).
