Starting implementation. Recon findings and design:

## What exists
- tests/macro_differential.rs already differentials macro *expansion* (interpreter vs compiled strategy-B component).
- Nothing differentials *execution*: tests/examples.rs locks docs/examples.json (84 snippets: 74 success + 10 error) against the interpreter only.
- The pieces for a compiled path all exist: build::build_files compiles a self-contained file in-process (no external toolchain), host::HostComponent instantiates and calls exports (pattern proven in tests/backend_numeric.rs), and the backend's to-string is F2's port of print_value — the same printer that produced each example's recorded "value" field.

## Design: tests/differential.rs
For every example in docs/examples.json:
1. Interpreter: wavelet::eval_snippet (the oracle).
2. Compiled: wrap the snippet into a component —
   - Package header (unless the snippet has one), declarations (Def/DefType/Import/Export/...) kept top-level,
   - expression forms sequenced into `Export {name: main result: string} Def main Fn {}` via Let-chaining, final expression wrapped in to-string(...),
   - build with build_files, instantiate with HostComponent, call main() -> string.
3. Assert: success examples must return exactly the recorded value string; error examples must also fail compiled (build error or trap).
4. An explicit skip table (example id -> reason) for examples the backend can't express yet (map/Fn lambdas, print/println, eval, DefMacro-in-snippet, ...). The table ratchets: a skipped example that starts agreeing fails the test with "remove it from the skip list". This is what makes the harness landable today and permanent (per 1.1: keep for the life of the project).
