## What it does
Every example in docs/examples.json runs through eval_snippet (the oracle)
AND a component built through the real emitter: declarations stay top-level,
expressions are sequenced into an exported `differential-main: func() ->
string` returning `to-string(Do [...])`; built in-process via
build::build_files, executed in the capability-free HostComponent. Agreement
is exact string equality (backend to-string is the print_value port);
error examples must fail on both sides. Complements
tests/macro_differential.rs (the expansion half of F4).

## The gate mechanics
21/84 agree today; the 63 divergent examples are SKIP-listed with the
reason and the plan slot that fixes each (6.2 F2 to-string fidelity: 37;
missing stdlib builtins 5.6/5.7: 16; pi 5.7: 2; flags 5.4: 2; expand
confinement 6.1.8: 1; runtime gradual checks goal 3: 2; overload dispatch: 3).
Skipped examples still execute — one that starts agreeing FAILS the suite
until its entry is deleted, so the list only ratchets down as goals 3-6 land.

## New finding surfaced by the harness
The build path (build_files) runs the pure checker but never
resolve_overloads, so a NON-exported overload set collapses to the
last-defined member and every call dispatches there regardless of argument
type — ty-overload-int/str return each other's values compiled, and
The-directed selection (ty-the-return) picks the wrong overload too.
Exported overload sets are unaffected (they mangle per member). Worth a
slot under goal 3 (3.2 is adjacent) or 5.6 if not already implied.
