## The one big call (veto here if you disagree)

The vault contains two giant initiatives that both rewrite emit.rs:

1. "Work to get to ABI wavelet" (total types, ABI-native representation,
   drop the dynamic core), and
2. "Replace the interpreter with the compiler" (compiled macros, REPL, run).

Plus the in-flight Round-trip Stage 3, whose remaining items mostly extend the
current *boxed* backend (f32, flags, stdlib parity as tag-switch builtins...).

These conflict: every hour spent extending or compiling the boxed dynamic
representation is thrown away when ABI wavelet replaces it, and
replace-interpreter is dramatically smaller *after* ABI wavelet (typed
tree->tree macros instead of compiling the full dynamic language). So the new
plan sequences: harnesses first -> core-language decisions (the drop list) ->
total type system -> WIT-surface completeness (representation-independent
halves: constructors, payload-less results, cross-package refs, resources) ->
ABI-native backend (where the representation halves of Stage 3 land, with
"conformance matrix fully green" as its exit gate) -> then retire the
interpreter from macros/REPL/run. This re-sequences the instruction on
lot:033i1kmtWX08Pj9GQWvMa6 (update 006) to finish Stage 3 in the current
backend first — remaining boxed-backend Stage 3 items are deliberately NOT
being finished in the boxed regime.

## Other key decisions being applied (details in the follow-up)

- "Implement garbage collection" folds into ABI wavelet's "Rethink memory
  management" — GC over today's tagged boxes would be undone by the header-less
  canonical layout, and wasm-GC can't hold Canonical-ABI linear-memory values
  anyway (at most it fits the closure residue).
- Replace-interpreter's "GC first" prerequisite is dropped as internally
  inconsistent: its own REPL design compiles a fresh instance per entry, so
  leaks are bounded per-entry; the memory story comes from the ABI work.
- Stdlib split appears three times (top-level Thing, F1, "monomorphize the
  higher-order stdlib") — merged into one goal inside the ABI phase, so the
  stdlib is written in Wavelet once, typed/monomorphic from the start.
- "Full u64 range" stops being a standalone design decision: it dissolves into
  typed numeric literals (type-system goal) + unboxed scalars (ABI goal).
- "Higher order functions as libraries", "Closures over the wire", and
  ABI's defunctionalization question are one decision; merged under a single
  "closure story" decision task (write-up only, per your original note).
- "Implement boundary coercions and a safely wrapper" contradicts ABI's "drop
  runtime The checks and checked coercions" — rescoped to a residual
  "safely for genuinely-dynamic data" and parked as unscheduled.
- Docs tasks parked under an unscheduled goal, per your instruction 002.
- The comprehensive code review is scheduled early but rescoped away from deep
  emit.rs polish (that file is rewritten by the ABI phase).

New top-level goal skeleton: 1 harnesses (differential F4 + conformance
custody), 2 shrink/settle the core (review, drop list, closure story, trim,
naming, .wlt), 3 total type system, 4 full WIT surface expressible,
5 ABI-native backend (matrix green = exit), 6 retire the interpreter from
execution surfaces, 7 build/CLI/tooling chores, 8 unscheduled (futures, parked
questions, docs).
