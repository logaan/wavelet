tests/generic_bridge.rs (an existing synthetic-WIT bridge suite):

- Wavelet can LIFT/LOWER dep-defined enum/variant/flags/list/option/result
  values and resource handles through imported interfaces (generic bridge),
  incl. constructor/method/static ops.
- But "Wavelet source has no enum/variant/flags type syntax to re-declare
  them" (their words) — DefType covers records (`DefType point {x: s32 y: s32}`)
  and list aliases (`DefType nums list(s32)`), not variant/enum/flags. So the
  values interface types must come from the vendored dep, and exporting into
  the foreign iface (`Export {name: ... iface: "roundtrip:suite/values"}`)
  must carry the signatures.
- Type text in source uses parens: `tuple(s32 string)`, `list(s32)`.
- Export record form: `Export {name: f params: {a: t} result: t}`; per-file
  package: `Package "demo:app@0.1.0"`.
- Build: `wavelet build src/file.wvl -o out` with vendored WIT at
