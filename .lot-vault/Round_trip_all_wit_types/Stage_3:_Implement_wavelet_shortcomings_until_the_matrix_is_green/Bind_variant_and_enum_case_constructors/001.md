# Bind variant and enum case constructors

Case names from a DefType variant are unbound ("unbound name `north`"), and
dep-defined cases are not reachable either ("`roundtrip:suite@0.1.0` does not
export `circle` in `types`"). Constructing variant/enum values is therefore
impossible; Match destructuring works. Also: list-form DefType always
synthesizes WIT `variant`, never `enum`. Acceptance: shape-rt, direction-rt,
option-shape-rt, result-tuple-direction-rt err side implementable in both
roles.
