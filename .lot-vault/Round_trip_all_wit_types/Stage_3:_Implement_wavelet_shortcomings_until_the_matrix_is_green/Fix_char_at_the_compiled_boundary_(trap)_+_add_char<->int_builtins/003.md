Done and ready for review: https://github.com/logaan/wavelet/pull/30 (draft,
branch worktree-fix-char, stacked on PR #29).

Both halves of the task landed:

1. Boundary trap fixed. Root cause: compiled chars box as TAG_CHAR, but the
   boundary lowered them through unbox_int (traps unless TAG_INT); lifting had
   the mirror bug (an incoming char boxed as an int, so eq(c 'a') was false in
   guest). Chars now lower via a dedicated unbox_char helper and lift back into
   TAG_CHAR boxes everywhere (flat args/results, list elements, option/result
   payloads); cmp_raw gained a char branch so lt/le/gt/ge order chars by
   codepoint instead of trapping; the meta-tree bridge's char-val wiring
   carries the char box on both sides.
2. char<->int conversions added, interpreter first: to-char(n) (int -> char,
   errors on surrogates / >0x10FFFF, char passes through) and the to-u8..to-s64
   family now accept a char and yield its codepoint. to-char is also
   implemented in the wasm backend with trap-where-interpreter-errors parity.
   Next-scalar transform is now expressible: to-char(add(to-u32(c) 1)).

Acceptance: char-rt restored in runner.wvl and green against both rust callees
(run/run-values/run-resources all `ok`); the callee char-rt transform is
expressible (backend to-u32-on-char rides on the "backend stdlib parity"
task #10, noted there in the PR).

Verification: new tests/backend_char.rs (5 tests, all fail pre-fix, all pass
post-fix); full cargo test green; examples regenerated (2 new stdlib docs
examples); LSP hover + completion updated; no lexer/grammar changes needed.
