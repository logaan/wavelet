# 4.7 Fix a set functor handle returned over a local-record element

The functor build path is otherwise at full interpreter parity, but an
export that returns the `set` handle over a locally-declared record element
forms a WIT interface cycle (the element's interface and the resource's
interface would each `use` the other), which WIT cannot express, so it is
rejected with a clear error. Lifting it needs the element record hoisted
into a shared types interface. emit.rs:7977-7988, dd-type-system.typ §6.
