Now load-bearing for the 2.3 decision (accepted 2026-07-03): expand-time
combinator templates are the *only* supported way a combinator defined in one
file is used from another (function types in cross-component signatures are
rejected in 5.8's first cut). So this Thing carries the "import map from
another file, call it with a local Fn" experience end-to-end, via the
existing macro-component import machinery. Ergonomic dependency: 7.4 (an
import that is both a runtime dependency and a macro library), since
combinator libraries will typically want to ship both.
