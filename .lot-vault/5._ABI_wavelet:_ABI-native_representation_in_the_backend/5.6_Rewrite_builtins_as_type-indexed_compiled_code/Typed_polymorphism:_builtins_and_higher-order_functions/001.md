# Typed polymorphism: builtins and higher-order functions

The dynamic core's polymorphism lives in the builtins: eq/compare work on any
tagged value, arithmetic dispatches on int-vs-dec tags at runtime, len works
on any sequence. Under ABI wavelet every call site must resolve to one
concretely-typed operation, because an unboxed value carries no tag to
dispatch on. Phase C's overload machinery is the tool: builtins become
overload sets resolved statically per call site.
