# 5.8 Implement the closure story decided in 2.3

Implement whichever closure story 2.3 decides. If defunctionalization: every Fn in a component becomes a case in a generated variant (discriminant = which function, payload = its typed captures) and application becomes one generated dispatch function per arrow type, making closures plain WIT variants that can cross component boundaries. If rejected: fence off one private non-ABI representation for closures only, confined inside the component (5.1's memory story must then keep headers for this one family). Absorbs the parity gap 'Support qualified symbols as bare values' (a bare alias/name value IS a function value, so its representation is this decision).
