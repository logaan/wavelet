# 3.11 Run the total checker ahead of the interpreter

The oracle relationship inverts for types: instead of the interpreter
defining behaviour that the checker must not preempt (the current rule —
"the checker must never preempt an existing runtime error"), the total
checker runs first and the interpreter only ever sees well-typed programs.
Runtime type-error paths in interp.rs become unreachable and are deleted;
what remains of the interpreter is the evaluation semantics both backends
share.
