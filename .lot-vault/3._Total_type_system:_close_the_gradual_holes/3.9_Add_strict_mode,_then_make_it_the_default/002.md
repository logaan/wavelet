**Question: what is the gate for flipping strict mode (`Type::Unknown` is an
error) from opt-in flag to the default?**

Options:

1. **Objective green gate (recommended).** Flip when the documented example
   suite (docs/examples.json → tests/examples.rs) and the conformance suite
   both pass under strict. That structurally requires the 2.2 drops to be
   implemented (the 3.10 burn-down), so the gate can't be gamed; until then
   strict ships opt-in and its failure list *is* the burn-down backlog,
   burned down category by category as planned.
2. **Phase gate.** Flip at the start of goal 5 regardless of suite state and
   fix fallout in place. Gets the pressure of a hard deadline at the cost of a
   long red main and a mixed-up burn-down.
3. **Flip immediately, keep a `--gradual` legacy flag.** Inverts the default
   before the drops exist; every existing program hits the wall at once and
   the legacy flag hardens into a de-facto second dialect.

Recommendation: **option 1**, with two riders: (a) from now on, new examples
and conformance programs are authored strict-clean, so the backlog only ever
shrinks; (b) once flipped, *delete* the gradual mode rather than leaving it as
a non-default flag — a lingering gradual dialect quietly undoes 2.2.6.
