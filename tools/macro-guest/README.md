# `wavelet-macro-guest` — the produced macro-component guest (Step 9)

The guest crate behind a **produced** `wavelet:meta/macros` component. When
`wavelet build` is given a Wavelet macro-library file (a `.wvl` whose top level
is `Package` + `DefMacro`s only — see `src/macrobuild.rs`), it compiles the file
into a component by building *this* crate against the file's source.

This is **strategy A: interpreter-in-a-component** (see
`dev-notes/macro-components/step-09-produce-macro-component.md`). Rather than
compile each macro body to a wasm function, the guest bundles the **Wavelet
interpreter** (it depends on the `wavelet` crate) plus the macro file's source as
embedded data, and runs the macros through `wavelet::macrolib` — the same
interpreter and the same `expand_once` the local ahead-of-time expander
(`wavelet::expand::expand_file`) uses. So a macro means exactly the same thing
expanded locally or through the produced component.

## How the producer drives it

`wavelet::macrobuild::build_macro_component` does, in process:

1. write the macro `.wvl` source to a temp file and set `WAVELET_MACRO_SRC` to
   its path;
2. `cargo build --release --target wasm32-unknown-unknown` in this crate
   (`build.rs` reads `WAVELET_MACRO_SRC` and embeds the source via
   `include_str!` of an `OUT_DIR` copy);
3. componentize the core module with `wit_component::ComponentEncoder` (the
   metadata wit-bindgen embeds is enough — **no `wasm-tools` shell-out**).

Built for **`wasm32-unknown-unknown` (no WASI)** so the component instantiates
under the consumer's empty, capability-free linker (Step 2). The `wavelet`
dependency is taken with `default-features = false`, which drops the
`playground` feature (`wasm-bindgen`) so the guest carries no
`__wbindgen_placeholder__` imports.

This crate is **standalone**: not part of the `wavelet` workspace, and never
built by `cargo test`. The dogfood test (`tests/produced_macros.rs`) consumes a
**prebuilt** component checked in at `tests/fixtures/produced-macros.wasm`, so
the suite stays hermetic.

## Regenerating `src/macro_lib.rs`

`src/macro_lib.rs` is generated by `wit-bindgen` from `wit/` and committed (so
the crate builds without re-running it). Regenerate after a `wit` change:

```console
cd tools/macro-guest
wit-bindgen rust wit --world macro-lib --generate-all --out-dir src
```

`wit/deps/wavelet-meta/code.wit` is a **vendored copy** of the repo's
`wit/meta/code.wit`; keep them in sync if the wire type changes.

Tool versions used: `wit-bindgen 0.43.0`, `wasm-tools 1.235.0` (only for manual
inspection), Rust target `wasm32-unknown-unknown`.
